How to Use the Tech Tinker Club Quiz

This quiz helps children review and strengthen ideas learned in Tech Tinker Club sessions. Each week covers a different topic from our Micro:bit projects – such as inputs and outputs, loops, conditions, variables, and sensors – with a mix of multiple-choice and drag-and-drop questions.

The goal is not to memorise facts, but to practise logical thinking. The questions encourage children to read pseudocode (a simplified way of writing code) and to predict what the Micro:bit would do step by step.

How to Take the Quiz

  • Choose a week: Each week matches a club topic. Start with one you’ve already covered.
  • Read carefully: Go line-by-line through any pseudocode before answering.
  • Answer: Click the best choice or drag terms into boxes for match-type questions.
  • Hints: Some questions include a hint – using it won’t affect your score.
  • Submit and review: Check your answer, read the feedback, and continue.
  • Repeat freely: The quiz doesn’t save data; it’s for open practice and exploration.

Understanding Pseudocode

Pseudocode is a simple, human-readable way of writing a program. It describes what the program does without worrying about exact punctuation or syntax. It sits halfway between everyday language and real code.

In MakeCode, you build programs from coloured blocks. The same logic in pseudocode might look like this:

repeat 3 times
    show icon "heart"
    pause 200
end

This means exactly the same as the blocks: show a heart, pause briefly, and repeat three times.

Why use pseudocode?

  • It helps children focus on logic rather than colours or shapes.
  • It builds transferable programming skills for text-based coding later on.
  • It improves debugging and reasoning – reading before running code.
  • It prepares pupils for KS3 and GCSE computing, where pseudocode is standard.

Tips for Working Through the Quiz

  • Take your time – logic is about reasoning, not speed.
  • Keep a pencil and paper to track changing values such as x or score.
  • Check conditions carefully (<, >, =) – small details matter.
  • Understand loops: repeat N times runs a set number; forever never stops.
  • Remember timing: 1000 ms = 1 second. Divide total time by pause length to estimate loops.
  • Relate pseudocode to MakeCode blocks – imagine which blocks would match each line.
  • Be curious: if you’re unsure, try the same logic later in MakeCode and see what happens.

Why Practising Logic Matters

The quiz is not about memorising commands. It’s about learning how computers think. By reasoning through loops, conditions, and variables, children develop computational thinking – the ability to break down a problem, predict outcomes, and debug mistakes.

These are core STEM skills that build confidence in maths, science, and creative problem-solving. Every question helps develop careful thinking and persistence – key qualities for young makers and programmers.
